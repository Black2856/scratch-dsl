  ## 1. Phase 0〜6完了後の現状

  現在の正本は Scratch 互換 DSL です。

  DSL
  → validator
  → HTMLJS Runtimeで実行
  → project.json serializer
  → SB3 packager
  → scratch-parser検証

  できること:

  - DSLの構造・参照・スコープ検証
  - headless Runtime実行
  - Canvas描画、入力、asset、簡易音声
  - clone、procedure、pen、monitorの最小動作
  - 検証済みDSLから .sb3 を生成
  - scratch-parser によるSB3形式検証

  重要な制約:

  - .sb3 はRuntime状態から生成せず、検証済みDSLから生成する
  - Runtime上のclone個体は保存しない
  - Scratch完全互換ではない
  - scratch-parser 通過は形式上の妥当性確認であり、Scratch公式エディタでの完全動作保証ではない
  - 既存SB3のimport・round-tripは未実装
  - 公式GUI、Paint/Sound editor、サイト共有は対象外

  直近のテスト実績:

  - Nodeテスト: 131件成功
  - Playwright: 9件成功
